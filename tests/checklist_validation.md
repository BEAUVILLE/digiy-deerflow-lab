# Checklist de validation — DIGIY DeerFlow Lab

Ce fichier sert à vérifier chaque production avant utilisation terrain.

Aucune fiche, page, offre, affiche, script ou sortie préparée par le Lab ne doit partir en public sans cette vérification.

---

## 1. Contrôle terrain

La sortie est-elle simple à comprendre ?

* [ ] Le texte est court.
* [ ] Le vocabulaire est clair.
* [ ] Un pro local peut comprendre sans explication compliquée.
* [ ] Le message donne envie d’agir.
* [ ] Il n’y a pas de blabla inutile.
* [ ] Le QR, le contact ou l’action sont visibles.
* [ ] La sortie peut être comprise sur téléphone.

Validation terrain :

```txt
OK / À corriger :
```

---

## 2. Contrôle DIGIYLYFE

La sortie respecte-t-elle l’esprit DIGIYLYFE ?

* [ ] Le terrain garde la main.
* [ ] Le paiement direct au pro est respecté.
* [ ] Aucune commission cachée n’est annoncée.
* [ ] Le ton est digne, clair et humain.
* [ ] La production sert le Club des Métiers du Terrain.
* [ ] La sortie ne transforme pas DIGIYLYFE en usine à gaz.
* [ ] La voix DIGIY reste la route principale côté client.
* [ ] DeerFlow reste en atelier, pas en façade publique.

Validation DIGIY :

```txt
OK / À corriger :
```

---

## 3. Contrôle données abonné / professionnel

Les données viennent-elles bien du professionnel concerné ?

* [ ] Le nom public est validé par le professionnel.
* [ ] Le métier est validé par le professionnel.
* [ ] Les zones d’intervention sont validées par le professionnel.
* [ ] Les services proposés sont validés par le professionnel.
* [ ] Les horaires éventuels sont validés par le professionnel.
* [ ] Le contact public est validé par le professionnel.
* [ ] Le message WhatsApp est validé par le professionnel.
* [ ] La fiche ne contient pas de données inventées.
* [ ] La fiche ne promet pas plus que ce que le pro peut réellement faire.

Règle : l’abonné renseigne ses propres données, car lui seul possède ses informations réelles terrain.

Validation données pro :

```txt
OK / À corriger :
```

---

## 4. Contrôle photos et médias

Les photos sont-elles propres et légères ?

* [ ] Aucune photo n’est stockée directement dans le fichier.
* [ ] Aucune image lourde n’est ajoutée dans le dépôt.
* [ ] La photo principale est renseignée en URL seulement.
* [ ] Les photos de galerie sont renseignées en URL seulement.
* [ ] Les URLs photos sont publiques ou accessibles.
* [ ] Les URLs photos sont validées par le professionnel.
* [ ] Les photos ne montrent pas de personnes sans accord.
* [ ] Les photos ne contiennent pas d’information privée visible.

Format recommandé :

```txt
photo_url: "https://..."
gallery_urls:
- "https://..."
- "https://..."
- "https://..."
```

Validation photos :

```txt
OK / À corriger :
```

* [ ] Aucune donnée client privée.

* [ ] Aucun slug interne sensible.
* [ ] Aucun code de connexion.
* [ ] Aucun PIN.
* [ ] Aucun détail qui expose la production.

## 6. Contrôle action

La sortie aide-t-elle vraiment à avancer ?

* [ ] Elle peut servir à vendre.
* [ ] Elle peut servir à expliquer.
* [ ] Elle peut servir à recruter un pro.
* [ ] Elle peut servir à orienter un client.
* [ ] Elle peut être envoyée par WhatsApp ou montrée sur téléphone.
* [ ] Elle donne une prochaine action claire.
* [ ] Elle respecte le contact direct entre client et pro.

## 7. Contrôle format

Le fichier est-il propre techniquement ?

* [ ] Le nom du fichier est clair.
* [ ] Le fichier est rangé dans le bon dossier.
* [ ] Le texte est lisible sur téléphone.
* [ ] Les liens fonctionnent.
* [ ] Les boutons ou appels à l’action sont compréhensibles.
* [ ] Le fichier peut être repris plus tard sans confusion.
* [ ] Le fichier ne mélange pas brouillon, production et données sensibles.
* [ ] Les URLs sont propres et testables.
